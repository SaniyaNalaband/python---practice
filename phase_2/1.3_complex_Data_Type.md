# 📖 1. What is a Complex Number?

A **complex number** consists of two parts:

```
Real Part + Imaginary Part
```

General form:

```
a + bj
```

Where:

- `a` = Real part
- `b` = Imaginary part
- `j` = Imaginary unit (`√-1`)

Examples:

```python
2 + 3j
```

```python
5 - 4j
```

```python
0 + 8j
```

```python
10 + 0j
```

---

# 🤔 2. Why Does Python Use `j` Instead of `i`?

In mathematics, the imaginary unit is usually written as **`i`**.

In Python, it is written as **`j`** because `i` is commonly used as a variable name in programming.

Correct:

```python
3 + 4j
```

Wrong:

```python
3 + 4i
```

This produces an error.

# 🪞 9. Complex Conjugate

Every complex number has a **conjugate**.

Example:

```
3 + 4j
```

Conjugate:

```
3 - 4j
```

Python provides the `conjugate()` method.

```python
z = 3 + 4j

print(z.conjugate())
```

Output:

```text
(3-4j)
```

---

# 📏 10. Magnitude (Absolute Value)

Use `abs()` to find the magnitude of a complex number.

```python
z = 3 + 4j

print(abs(z))
```

Output:

```text
5.0
```

This is calculated using the Pythagorean theorem.

# ⚠️ 11. Common Mistakes

## ❌ Using `i` Instead of `j`

Wrong:

```python
5 + 2i
```

Correct:

```python
5 + 2j
```

---

## ❌ Forgetting the Number Before `j`

Wrong:

```python
j
```

Correct:

```python
1j
```

---

## ❌ Treating `.real` as an Integer

```python
z = 5 + 6j

print(z.real)
```

Output:

```text
5.0
```

`.real` returns a **float**, even if the real part looks like an integer.

---

# 💡 12. Best Practices

✅ Use meaningful variable names:

```python
voltage = 230 + 15j
```

Instead of:

```python
x = 230 + 15j
```

---

✅ Use `.real` and `.imag` when you need the individual parts.
