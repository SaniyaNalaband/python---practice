# 📖 1. `add()`

Adds **one element** to a set.

### Syntax

```python
set.add(element)
```

---

## Example 1

```python
fruits = {"Apple", "Banana"}

fruits.add("Mango")

print(fruits)
```

Possible Output

```text
{'Apple', 'Banana', 'Mango'}
```

---

## Example 2

```python
numbers = {10, 20}

numbers.add(30)

print(numbers)
```

Possible Output

```text
{10, 20, 30}
```

---

## Adding a Duplicate

```python
numbers = {10, 20}

numbers.add(20)

print(numbers)
```

Output

```text
{10, 20}
```

Nothing changes because sets do not allow duplicate values.

---

## Add Different Data Types

```python
data = {"Python"}

data.add(100)

data.add(True)

print(data)
```

Possible Output

```text
{True, 100, 'Python'}
```

---

## Add a Tuple

```python
items = {"Apple"}

items.add(("Banana", "Mango"))

print(items)
```

Possible Output

```text
{'Apple', ('Banana', 'Mango')}
```

A tuple is immutable, so it can be stored in a set.

---

## Cannot Add a List

```python
items = {"Apple"}

items.add(["Banana", "Mango"])
```

Output

```text
TypeError: unhashable type: 'list'
```

Lists are mutable, so they cannot be elements of a set.

---

# 📖 2. `update()`

Adds **multiple elements** from another iterable.

### Syntax

```python
set.update(iterable)
```

---

## Example 1

```python
numbers = {1, 2}

numbers.update([3, 4])

print(numbers)
```

Possible Output

```text
{1, 2, 3, 4}
```

---

## Example 2 (Using Another Set)

```python
a = {1, 2}
b = {3, 4}

a.update(b)

print(a)
```

Possible Output

```text
{1, 2, 3, 4}
```

---

## Example 3 (Using a Tuple)

```python
numbers = {1, 2}

numbers.update((3, 4))

print(numbers)
```

Possible Output

```text
{1, 2, 3, 4}
```

---

## Example 4 (Using a String)

```python
letters = {"A"}

letters.update("BCD")

print(letters)
```

Possible Output

```text
{'A', 'B', 'C', 'D'}
```

Each character becomes a separate element.

---

## Duplicate Values

```python
numbers = {1, 2}

numbers.update([2, 3, 4])

print(numbers)
```

Output

```text
{1, 2, 3, 4}
```

The duplicate value `2` is ignored.

# 📊 `add()` vs `update()`

| Feature | `add()` | `update()` |
|---------|----------|------------|
| Adds one element | ✅ Yes | ❌ No |
| Adds multiple elements | ❌ No | ✅ Yes |
| Accepts iterable | As one element | Expands the iterable |
| Ignores duplicates | ✅ Yes | ✅ Yes |

---

### Example Comparison

```python
a = {1, 2}

a.add((3, 4))

print(a)
```

Possible Output

```text
{1, 2, (3, 4)}
```

The tuple is added as **one element**.

---

```python
a = {1, 2}

a.update((3, 4))

print(a)
```

Possible Output

```text
{1, 2, 3, 4}
```

The tuple is unpacked, and each value is added separately.

# ⚠️ Common Mistakes

## ❌ Trying to Add Multiple Elements with `add()`

```python
numbers = {1, 2}

numbers.add(3, 4)
```

Output

```text
TypeError
```

Correct:

```python
numbers.update([3, 4])
```

---

## ❌ Trying to Add a List

```python
items = {"Apple"}

items.add(["Banana"])
```

Output

```text
TypeError
```

---

## ❌ Expecting Duplicates

```python
numbers = {1, 2}

numbers.add(2)

print(numbers)
```

Output

```text
{1, 2}
```

Sets automatically ignore duplicate values.

---

# 💡 Best Practices

- Use `add()` for a single element.
- Use `update()` for multiple elements.
- Remember that duplicates are ignored automatically.
- Only immutable objects (like tuples) can be elements of a set.

---

# 🚀 Pro Tips

You can update a set using multiple iterables at once.

```python
numbers = {1}

numbers.update([2, 3], (4, 5), {6, 7})

print(numbers)
```

Possible Output

```text
{1, 2, 3, 4, 5, 6, 7}
```

---

# 🧠 Memory Trick

```text
add()

↓

One Element

----------------

update()

↓

Many Elements

↓

Expands Iterable
```
