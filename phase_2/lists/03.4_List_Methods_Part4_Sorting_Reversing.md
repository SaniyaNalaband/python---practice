# 📖 Why Do We Need Sorting?

Suppose you have student marks.

```python
marks = [85, 70, 95, 60, 90]
```

You may want them in ascending order.

Python provides `sort()`.

---

# 📖 1. `sort()`

Sorts the original list in **ascending order**.

### Syntax

```python
list.sort()
```

---

## Example 1

```python
numbers = [40, 10, 30, 20]

numbers.sort()

print(numbers)
```

Output

```text
[10, 20, 30, 40]
```

---

## Example 2 (Strings)

```python
fruits = ["Mango", "Apple", "Banana"]

fruits.sort()

print(fruits)
```

Output

```text
['Apple', 'Banana', 'Mango']
```

Strings are sorted alphabetically.

---

# 📖 Descending Order

Use:

```python
reverse=True
```

```python
numbers = [40, 10, 30, 20]

numbers.sort(reverse=True)

print(numbers)
```

Output

```text
[40, 30, 20, 10]
```

---

# 📖 Sort Ignoring Case

```python
names = ["saniya", "Rahul", "aisha", "Kiran"]

names.sort(key=str.lower)

print(names)
```

Output

```text
['aisha', 'Kiran', 'Rahul', 'saniya']
```

Without `key=str.lower`, uppercase words come before lowercase ones because sorting is based on ASCII/Unicode values.

---

# 📖 Sort by Length

```python
words = ["Python", "AI", "Machine", "ML"]

words.sort(key=len)

print(words)
```

Output

```text
['AI', 'ML', 'Python', 'Machine']
```

---

# 📖 2. `reverse()`

Reverses the current order of the list.

### Syntax

```python
list.reverse()
```

---

## Example

```python
numbers = [10, 20, 30, 40]

numbers.reverse()

print(numbers)
```

Output

```text
[40, 30, 20, 10]
```

**Important:** `reverse()` does **not sort** the list. It only flips the current order.

---

Example:

```python
numbers = [30, 10, 40, 20]

numbers.reverse()

print(numbers)
```

Output

```text
[20, 40, 10, 30]
```

---

# 📖 3. `sorted()`

`sorted()` is a **built-in function**, not a list method.

It returns a **new sorted list** without changing the original.

### Syntax

```python
sorted(iterable)
```

---

## Example

```python
numbers = [30, 10, 40, 20]

new_list = sorted(numbers)

print(new_list)
print(numbers)
```

Output

```text
[10, 20, 30, 40]
[30, 10, 40, 20]
```

Notice:

- `numbers` is unchanged.
- `new_list` is sorted.

---

## Descending Order

```python
numbers = [30, 10, 40, 20]

print(sorted(numbers, reverse=True))
```

Output

```text
[40, 30, 20, 10]
```

# 📊 `sort()` vs `sorted()` vs `reverse()`

| Feature | `sort()` | `sorted()` | `reverse()` |
|---------|----------|------------|-------------|
| Type | List Method | Built-in Function | List Method |
| Returns | `None` | New List | `None` |
| Changes Original | ✅ Yes | ❌ No | ✅ Yes |
| Sorts | ✅ Yes | ✅ Yes | ❌ No |
| Reverses Order | ❌ | ❌ | ✅ Yes |

# ⚠️ Common Mistakes

## ❌ Saving `sort()`

```python
numbers = [3, 1, 2]

result = numbers.sort()

print(result)
```

Output

```text
None
```

Correct:

```python
numbers.sort()

print(numbers)
```

---

## ❌ Thinking `reverse()` Sorts

```python
numbers = [30, 10, 40]

numbers.reverse()

print(numbers)
```

Output

```text
[40, 10, 30]
```

It only reverses the existing order.

---

# 💡 Best Practices

- Use `sort()` when you want to modify the original list.
- Use `sorted()` when you need to keep the original list unchanged.
- Use `reverse()` only to reverse the current order.

---

# 🚀 Pro Tips

Sort tuples by the second element:

```python
students = [
    ("Rahul", 85),
    ("Saniya", 95),
    ("Aisha", 90)
]

students.sort(key=lambda x: x[1])

print(students)
```

Output

```text
[('Rahul', 85), ('Aisha', 90), ('Saniya', 95)]
```

# 🧠 Memory Trick

```
sort()

↓

------------------

sorted()

↓

New Sorted List

------------------

reverse()

↓

Flip Current Order
```
