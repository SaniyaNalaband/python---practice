# 📖 1. `remove()`

Removes a specified element.

### Syntax

```python
set.remove(element)
```

If the element does not exist, Python raises a `KeyError`.

---

## Example 1

```python
fruits = {"Apple", "Banana", "Mango"}

fruits.remove("Banana")

print(fruits)
```

Possible Output

```text
{'Apple', 'Mango'}
```

---

## Example 2 (Element Not Found)

```python
numbers = {10, 20, 30}

numbers.remove(50)
```

Output

```text
KeyError: 50
```

---

# 📖 2. `discard()`

Removes a specified element.

### Syntax

```python
set.discard(element)
```

Unlike `remove()`, if the element is not present, **no error is raised**.

---

## Example 1

```python
numbers = {10, 20, 30}

numbers.discard(20)

print(numbers)
```

Possible Output

```text
{10, 30}
```

---

## Example 2 (Element Not Found)

```python
numbers = {10, 20, 30}

numbers.discard(50)

print(numbers)
```

Output

```text
{10, 20, 30}
```

Nothing happens.

---

# 📖 3. `pop()`

Removes and returns **an arbitrary element** from the set.

### Syntax

```python
set.pop()
```

⚠️ Since sets are unordered, you **cannot predict** which element will be removed.

---

## Example

```python
languages = {"Python", "Java", "SQL"}

removed = languages.pop()

print("Removed:", removed)
print("Remaining:", languages)
```

Possible Output

```text
Removed: Java
Remaining: {'Python', 'SQL'}
```

or

```text
Removed: SQL
Remaining: {'Python', 'Java'}
```

The removed element may vary between runs.

---

## Empty Set

```python
items = set()

items.pop()
```

Output

```text
KeyError: 'pop from an empty set'
```

---

# 📖 4. `clear()`

Removes **all elements** from the set.

### Syntax

```python
set.clear()
```

---

## Example

```python
numbers = {1, 2, 3}

numbers.clear()

print(numbers)
```

Output

```text
set()
```

The set still exists, but it is empty.

# 📊 Comparison Table

| Method | Removes By | Returns Removed Element? | Error If Missing? |
|---------|------------|--------------------------|-------------------|
| `remove()` | Value | ❌ No | ✅ `KeyError` |
| `discard()` | Value | ❌ No | ❌ No |
| `pop()` | Arbitrary element | ✅ Yes | ✅ On empty set |
| `clear()` | All elements | ❌ No | ❌ No |

# ⚠️ Common Mistakes

## ❌ Using `remove()` Without Checking

```python
numbers = {1, 2}

numbers.remove(5)
```

Raises

```text
KeyError
```

Safe approach:

```python
if 5 in numbers:
    numbers.remove(5)
```

---

## ❌ Assuming `pop()` Removes the "First" Element

```python
letters = {"A", "B", "C"}

letters.pop()
```

Sets have **no first or last element**.

---

## ❌ Thinking `clear()` Deletes the Variable

```python
numbers = {1, 2}

numbers.clear()

print(numbers)
```

Output

```text
set()
```

The variable still exists.

---

# 💡 Best Practices

- Use `remove()` when you're sure the element exists.
- Use `discard()` when you're not sure.
- Use `pop()` only when you don't care which element is removed.
- Use `clear()` to empty a set quickly.

---

# 🚀 Pro Tips

Safely remove an element:

```python
languages = {"Python", "Java"}

languages.discard("C++")

print(languages)
```

No error is raised.
